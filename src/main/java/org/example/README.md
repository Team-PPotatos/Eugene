## 프로젝트 요구사항

```
1. 은행(Bank)에서는 랜덤한 계좌번호를 만들 수 있다. ex) 00000-00000

2. 은행에서는 유저로부터 이름, 비밀번호 를 받고, 잔고는 0으로, 
   그리고 랜덤으로 생성한 계좌번호를 넣어 서비스 계층의 Service 기본 
   클래스로 전달해 은행에서 생성한 Array[i]에 저장할 수 있다.

3. 유저(User)는 은행을 통해 비밀번호 변경, 입금, 출금과 같은 업무를 볼 수 있다.

4. 은행은 유저로 부터 값을 받아, 서비스시스템(Service)에게 전달하여 
   응답에 대한 적절한 반응을 print해준다.  

5. 유저에게는 은행 버튼이 있으며 버튼을 누르고 값을 입력하면 은행에서 
   처리해 출력만 확인이 가능하다.

6. 은행에서는 유저로 부터 계좌번호를 입력받아 for문을 통해 account[i] 서비스의 getAccountnumber을 통해
   this.계좌가 내가 입력한 계좌가 존재하는지 확인 후 인덱스 를 찾아 Service로 부터 받은 값을 토대로 맞는 반응을 
   유저에게 전달한다.

7. 서비스에서는 은행으로부터 해당 인덱스의 값을 받아 해당 계좌번호의 비밀번호가 일치하는지 getpassword로 확인한 후,
   맞을 시에만 입금 혹은 출금의 행위를 한 후 그 결과를 은행에게 반환합니다.
   
8. 은행이 유저로부터 계좌를 받고, 계좌가 존재하는 지 확인 후 서비스에게 전달하면, 
   서비스가 비밀번호를 확인 후 맞을 시 0보다 큰 금액인지 확인하여 아닐 경우 행위를 실행하지 않는다 
   
9. 위와 마찬가지로 은행이 유저로부터 계좌를 받아 계좌가 존재하는지 확인하고
   출금 금액을 받고, 서비스가 비밀번호를 확인 후  현재 잔고보다 큰 금액일 경우 행위를 실행하지 않고, 
   출금이 가능한 경우 현재의 금액을 출금 후 금액으로 만든 후 현재 
   잔고를 print 혹은 반환한다.
   
10. 비밀번호를 바꾸는 경우, 은행이 유저로부터 계좌를 받아 서비스로부터
    해당 계좌가 존재하는지 확인한 후, 있을시 은행이 비밀번호를 받아
    서비스에게 전달한다. 서비스는 비밀번호가 일치하는지 확인 후 값을 받아
    그 후 유저가 바꾸려는 비밀번호를 입력 받고, 서비스는 해당 비밀번호로 변경해준 뒤
    결과를 print 혹은 반환한다.

11. 계좌와 관련된 행위(계좌 인덱스 찾기)는 은행에서 모두 처리한다.

12. 유저는 Do while 문으로 돌아가고, 5번을 입력하면 프로그램이 종료된다.

13. 유저는 5가지 버튼을 볼 수 있으며, 1번은 계좌 생성, 2번은 입금, 3번은 출금, 4번은 비밀번호 변경, 5번은 프로그램 종료 입니다.

14. 다른 번호를 입력했을 경우, 없는 업무를 선택했다는 메시지와 함께 다시 메뉴를 띄웁니다.
```

즉, 정말 간단하게 유저가 `main`, 은행이 `Controller` 서비스가 `Service` 계층이라고 생각하시면 됩니다.

![img.png](..%2F..%2F..%2F..%2F..%2F..%2FBank%2FBank%2Fsrc%2Fmain%2Fjava%2Forg%2Fexample%2Fimg.png)


- `Service`

```
속성 : accountNumber, Password, Budget, name

메서드 : getBudget, getPassword, setPassword, deposit, withdraw, getAccountNumber
```

- `Bank`

```
속성 :  index, accountNumber, account

메서드: checkAccountnumber(계좌의 인덱스를 리턴), findAccount(check머시기로 부터 받은 인덱스가 -1인지 아닌지를 확인 후 계좌가 존재하는지 아닌지 리턴),
 changePassword, createAccount, moneyOut, moneyIn
```

다이어그램에 있는 나머지 메서드는 구현 안하셔도 됩니다. 필요하다면 추가하셔도 됩니다.
